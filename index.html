Discover Lincoln: A Tourism Hub

Discover Lincoln is a comprehensive tourism and events website designed to highlight the Town of Lincoln’s attractions, businesses, and events. Built with user experience at its core, the site features an interactive map with robust filtering functionality, enabling users to explore Lincoln's offerings seamlessly. The site goes beyond basic browsing by incorporating a Lincoln Guessr game, offering a gamified experience for users to discover the town's landmarks in a fun, interactive way.

An accounts dashboard with user authentication allows users to register and log in to manage their own events, attractions, and businesses. The platform facilitates user-submitted content, which undergoes an approval process via Strapi CMS, ensuring quality control and consistency across the site. By linking businesses to events and attractions, the platform creates a dynamic and interconnected browsing experience.

Technologies and Tools:

  • Back-End: Node.js, Strapi CMS
  • Front-End: JavaScript, HTML (via Next.js), CSS (via Styled-jsx), React, Next.js
  • Map Integration: Leaflet.js
  • Development & Collaboration: GitHub, Visual Studio Code
  • Design & Prototyping: Figma, Adobe (Photoshop, Illustrator)

Development Process & Methodologies

Agile & Scrum Methodologies

The development process followed Agile principles, with work organized into sprints to ensure continuous progress. Scrum practices were implemented to track milestones, conduct regular reviews, and adapt to evolving requirements, ensuring that key features like the Lincoln Guessr game, user authentication, and dashboard management were effectively prioritized and delivered.

Testing

Comprehensive testing strategies were applied throughout the development process. Unit testing was used to verify core functionalities, particularly for key features like content submission, filtering, and authentication. Manual testing ensured that the platform was fully mobile-responsive and provided a seamless experience across devices, including the interactive map, account dashboard, and the Lincoln Guessr game.

Figma Design Process (User Stories, Site Map, Wireframes, Prototype)

User Stories

User stories were developed to prioritize key user experiences, focusing on tourists, business owners, and event planners.

Site Map

A detailed site map was created to ensure smooth navigation and a clear content structure for users. Events, attractions, and businesses were seamlessly integrated into the interactive map and linked dynamically across pages, enhancing the user’s journey through the site.

Wireframes

Low-fidelity wireframes were created to outline the structure and layout of all key pages, including the landing page, events page, attractions page, businesses page, and Lincoln Guessr. Special emphasis was placed on ensuring the site’s responsive design, allowing the interactive map, Lincoln Guessr, and dashboard features to function smoothly across both mobile and desktop devices.

Prototype (Desktop & Mobile)

High-fidelity prototypes were developed for both desktop and mobile views, ensuring full responsiveness. These prototypes covered all key pages and features, including the landing page, interactive map, events, attractions, and businesses pages, details pages, dashboard management, and Lincoln Guessr game, as well as the content submission workflows. This ensured a cohesive user experience regardless of the device used, with special attention given to the details of how users would navigate between content-rich pages and interactive features.

Feature Demonstrations

Landing Page

The landing page introduces Discover Lincoln with a clean navigation bar offering quick access to Maps, Events, Attractions, Businesses, and more. The hero section showcases the mission, inviting users to explore Lincoln. Smooth scrolling takes users through featured sections (events, attractions, businesses), each with a carousel of cards containing key details like dates, locations, and descriptions. The Lincoln Guessr section features a leaderboard and play button. The footer includes contact details and social media links.

Map Page

The interactive map of Lincoln, Ontario allows users to explore businesses, attractions, and events. Categories can be filtered, and the search bar provides quick access to specific locations. Clicking an icon opens a pop-up with essential details like date, time, and location, leading to a full details page for further information.

Events Page + Event Details Page

The Events page offers a comprehensive list of upcoming events. Users can filter by type, date, or sort alphabetically. Event cards display images, titles, and locations, with a “Learn More” button leading to detailed event pages that include a photo carousel, event description, organizer details, and accessibility information. A share button allows easy event promotion.

Attractions Page + Attraction Details Page

The Attractions page highlights Lincoln’s main attractions. Users can filter by type or search for specific locations. Attraction cards feature images, titles, and hours, leading to details pages with photo carousels, descriptions, contact info, and accessibility features like parking and wheelchair access.

Businesses Page + Business Details Page

The Businesses page lists local businesses with cards featuring images, titles, locations, and hours. Users can search and filter businesses by type or alphabetically. Clicking on a card opens the business details page, which includes photos, descriptions, contact info, and accessibility details like vegetarian options. A share button facilitates easy sharing.

About Page

The About page provides an overview of Lincoln’s history and identity, using scenic backgrounds and cards to showcase the town’s cultural and natural assets. The page concludes with an “Explore Lincoln” button that brings users back to the landing page.

LincolnGuessr Game Page

Lincoln Guessr is a fun, gamified exploration tool where users guess locations within Lincoln using Google Street View. Players earn points based on the accuracy of their guesses across three rounds, with scores displayed on a leaderboard. The game includes a help section and the option to exit back to the landing page at any time.

Account Registration + Login Pages

The registration and login pages offer a secure, user-friendly experience, with a scenic Lincoln GIF background. New users register with email, username, and password, and upon logging in, access their dashboard to manage events, attractions, and businesses.

Account Dashboard + Create Pages

The dashboard allows users to create, edit, and delete posts for events, attractions, and businesses. Users fill out a form with key details, upload photos, and submit posts, which appear in the dashboard as “Pending” until approved by an admin through Strapi. Once approved, posts become visible on the respective pages, and users can edit or delete them as needed.

Strapi Integration Breakdown

Strapi CMS served as the backbone of the content management system for Discover Lincoln, managing user-generated content for events, businesses, and attractions. It efficiently structured and scaled content, leveraging relationships between content types to ensure smooth approvals and updates.

Content Management and Data Structures

Strapi managed various content types, including events, businesses, attractions, and about page information. Each content type was structured with flexible data models featuring fields like titles, dates, locations, descriptions, and media, keeping content organized and easy to manage. Relationships between content types—such as linking businesses to events—enhanced navigation and created dynamic content flows.

Content Creation, Approval, and Relationships

Strapi’s relational data structure allowed content types to interconnect, creating a seamless user experience. For example, an event could be linked to a specific business or attraction, enriching content discovery and encouraging users to explore Lincoln’s offerings more deeply through related content.

The approval workflow enabled admins to control what appeared on the site. Community-submitted content, such as events and businesses, was reviewed and approved through Strapi, maintaining quality and keeping the platform user-driven.

Customization and Flexibility

Strapi's Content-Type Builder allowed the addition of custom fields tailored to specific project needs, such as accessibility tags for businesses and events, ensuring inclusivity. This level of customization made the platform adaptable for future requirements without heavy redevelopment.

Media Management

Strapi’s Media Library managed all images and media files for events, attractions, and businesses. This centralized system made uploading, managing, and reusing visuals across the site simple, ensuring consistent presentation. Each image was linked to its relevant content type, such as About Page Info or Attractions, streamlining content management.

Code Highlights

Custom Interactive Map with Dynamic Markers

Code Snippet: MapPageMap.js

Challenge: Integrate dynamic, filterable markers on a Leaflet.js map within a React application, using custom icons and ensuring smooth performance.

Solution:

  • Custom Markers with Icons: Used 'ReactDOMServer.renderToString' to convert React elements (FontAwesome icons) into HTML strings for Leaflet's 'DivIcon', allowing custom-styled map icons.
  • Dynamic Filtering: Employed 'useEffect'  and 'useState'  hooks to update displayed markers based on user-selected filters from the context.
  • Performance Optimization: Managed state efficiently to prevent unnecessary re-renders, ensuring smooth performance with many markers.

Reflection: Gained experience integrating React with Leaflet.js and optimizing state management to improve performance and responsiveness.

Advanced Filtering Mechanism

Code Snippet: MapFilter.js

Challenge: Create an intuitive filtering system with individual category selection and a "Select All" option, updating the map accordingly.

Solution:

  • State Synchronization: Managed selection states using 'useState' for both individual filters and "Select All" options.
  • Hierarchical Logic: Implemented functions to handle individual and group selections, keeping filters synchronized across the app.
  • User Experience: Designed the UI for immediate feedback, enhancing usability.

Reflection: Improved skills in managing complex state logic and ensuring the UI remains user-friendly and responsive to changes.

Lincoln Guessr Game Map Component

Code Snippet: LincolnGuessrMap.js

Challenge: Develop an interactive map for the Lincoln Guessr game where users select locations, compare guesses to actual locations, and display connections.

Solution:

  • Interactive Map Events: Utilized 'useMapEvents'  from React Leaflet to handle user clicks for map guessing.
  • Custom Icons and Polylines: Created custom markers and polylines to connect guessed and actual locations, enhancing visual clarity.
  • Conditional Rendering: Implemented conditional logic to adjust map elements based on the game state.

Reflection: Developed real-time interactive features and learned the importance of providing visual feedback to enhance the user experience.

Conclusion

Discover Lincoln combines thoughtful UI design with robust front-end and back-end functionality to create an engaging platform for showcasing the Town of Lincoln. Using tools like Next.js, Leaflet.js, and Strapi CMS, the website provides dynamic content management, real-time updates, and smooth user navigation. The back-end, managed through Strapi, supports user authentication, content submission, and approval workflows, making the platform adaptable and scalable.

Extensive testing ensured that the site is fully responsive across both desktop and mobile. A comprehensive design process from wireframes to high-fidelity prototypes guided the development of features like the interactive map, content filtering, and user dashboards. The back-end setup also facilitated secure data management and content workflows.

This project highlights skills in front-end and back-end development, UI/UX design, content management, and testing, demonstrating the ability to build a flexible and scalable platform that can handle future updates and content growth.
